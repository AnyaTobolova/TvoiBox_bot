# Soft Launch Mini App

## Цель

Мягко запустить `mini app` на ограниченном круге людей, не ломая рабочий процесс в основном Telegram-боте и не переводя всех пользователей сразу.

## Первый круг запуска

- тренер;
- 2-3 лояльных клиента, которые готовы дать обратную связь;
- только сценарии записи, подтверждения, переноса, отмены и открытия календаря.

## Контур запуска

- внешний `dev`-контур `https://app.anyatobolova.ru/`;
- `@testtvoy_bot` как точка входа в mini app;
- основной production-бот остаётся резервным каналом на весь soft launch.

## Сценарий включения

1. Тренер и выбранные клиенты получают ссылку/инструкцию на вход через `@testtvoy_bot`.
2. Все новые тестовые действия в рамках soft launch проводятся только через mini app.
3. Если пользователь упирается в сбой, он возвращается в основной бот без ожидания фикса "на месте".
4. Все замечания фиксируются в `Этапы разработки.md` с датой, ролью (`клиент` / `тренер`) и фактическим сценарием.

## Что считаем обязательным в soft launch

- mini app открывается в Telegram без VPN;
- клиент может создать запись;
- тренер видит заявку и может подтвердить/отклонить/предложить перенос;
- отменённые и прошедшие тренировки не засоряют активные списки;
- `.ics` открывается/отдаётся на телефоне;
- fallback в основной бот остаётся доступным.

## Что считаем допустимым ограничением на этом этапе

- в Google Calendar UI тренера напоминание может визуально оставаться `30 минут`, даже если событие по API уже содержит `1 день` и `1 час`;
- это ограничение текущей схемы через `service account`, а не блокер soft launch.

## Правило отката

- если возникает критичный сбой в mini app, новые действия для soft launch останавливаются;
- пользователи переводятся обратно на основной бот;
- баг фиксируется сначала в `dev`, потом снова перепроверяется на ограниченном круге.

## Критерий завершения soft launch

- тренер может пользоваться mini app параллельно с ботом без блокирующих ошибок;
- выбранные клиенты проходят ключевые сценарии без ручного сопровождения на каждом шаге;
- остаются только некритичные UX-замечания, а не блокеры потока записи.
